简介

本文介绍Rust语言的零成本抽象在嵌入式系统中的应用。利用Rust语言的零成本抽象，可以实现高效的内存管理、并发编程和错误处理，从而提高嵌入式系统的性能和可靠性，同时不损失性能。

本文内容以及程序位于此仓库

在阅读本文前，建议先阅读rust embedded example book,了解如何使用Rust构建嵌入式项目。

工程介绍

源码介绍

源码如下：

#![no_main]
#![no_std]

use cortex_m::asm;
use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};
use stm32f303pac as pac;

/// 启用GPIOE时钟
/// 不使用inline。默认情况下编译器会将此函数优化为inline，不方便查看汇编代码
/// 设置no_mangle,不改变链接符号
#[inline(never)]
#[unsafe(no_mangle)]
fn enable_gpioe_clock(dp: &pac::Peripherals) {
    dp.rcc.ahbenr().modify(|_, w| w.iopeen().set_bit());
}

#[entry]
fn main() -> ! {
    // 初始化RTT打印
    rtt_init_print!();
    rprintln!("Starting simple LED blink program");

    // 获取外设访问
    let dp = pac::Peripherals::take().unwrap();

    // 启用GPIOE时钟
    enable_gpioe_clock(&dp);

    // 配置PE8为推挽输出模式
    let gpioe = &dp.gpioe;
    gpioe
        .moder()
        .modify(|_, w| unsafe { w.moder8().bits(0b01) }); // 设置为输出模式
    gpioe.otyper().modify(|_, w| w.ot8().clear_bit()); // 推挽输出

    rprintln!("LED blink initialized on PE8");

    // 主循环，周期性切换LED状态
    loop {
        // 打开LED (设置高电平)
        gpioe.bsrr().write(|w| w.bs8().set_bit());
        rprintln!("LED ON");
        asm::delay(1_000_000); // 延时

        // 关闭LED (设置低电平)
        gpioe.bsrr().write(|w| w.br8().set_bit());
        rprintln!("LED OFF");
        asm::delay(1_000_000); // 延时
    }
}

重点关注enable_gpioe_clock函数。下面代码中的#[inline(never)]要求编译器不要将该函数优化为内连函数，若被优化为内连函数，我们生成反汇编文件后得到的汇编文件中将没有这个函数的链接符号，函数内容将展开到main函数中，无法很好地定位到这个函数。#[unsafe(no_mangle)告诉编译器将函数的链接符号名设置为enable_gpioe_clock。这样便能很好地定位到这个函数对应的汇编码。

/// 启用GPIOE时钟
/// 不使用inline。默认情况下编译器会将此函数优化为inline，不方便查看汇编代码
/// 设置no_mangle,不改变链接符号
#[inline(never)]

本节将介绍函数 enable_gpioe_clock 控制寄存器的过程。

获取抽象设备

分析 Peripherals 结构

此结构体包含了众多芯片外设。

take() 函数的实现

进一步查看 take() 函数：

#![allow(unused)]
fn main() {
pub fn take() -> Option<Self> {
    critical_section::with(|_| {
        if unsafe { DEVICE_PERIPHERALS } {
            return None;
        }
        Some(unsafe { Peripherals::steal() })
    })
}
}

函数说明：

  • 返回 Option<Self>，其中 Self 是 Peripherals 结构体
  • 如果 DEVICE_PERIPHERALS 为 true，则返回 None
  • 否则返回 Some(unsafe { Peripherals::steal() })
  • 运行时将屏蔽所有中断，第一次运行后会将 DEVICE_PERIPHERALS 设置为 true，从而避免再次调用 take()

steal() 函数的实现

进一步查看 Peripherals::steal() 函数：

#![allow(unused)]
fn main() {
pub unsafe fn steal() -> Self {
    unsafe {
        DEVICE_PERIPHERALS = true;
        Peripherals {
            gpioa: Gpioa::steal(),
            gpiob: Gpiob::steal(),
            gpioc: Gpioc::steal(),
            gpiod: Gpiod::steal(),
            // ...
        }
    }
}
}

零成本抽象的体现

进一步查看 Gpioa::steal()：

#![allow(unused)]
fn main() {
pub unsafe fn steal() -> Self {
    Self {
        _marker: marker::PhantomData,
    }
}
}

重要发现： Gpioa 是一个没有意义的结构体，是不是很惊讶？我们已经接触到了 Rust 语言的 零成本抽象（zero abstraction）。

RegisterBlock 结构

指针和解引用实现

当我们继续查看 pub struct Periph<RB, const A: usize> 可以发现：

#![allow(unused)]
fn main() {
// crates/stm32f303pac/src/generic.rs
impl<RB, const A: usize> Periph<RB, A> {
    #[doc = "Pointer to the register block"]
    pub const PTR: *const RB = A as *const _;

    #[doc = "Return the pointer to the register block"]
    #[inline(always)]
    pub const fn ptr() -> *const RB {
        Self::PTR
    }
    // ...
}

impl<RB, const A: usize> core::ops::Deref for Periph<RB, A> {
    type Target = RB;

    #[inline(always)]
    fn deref(&self) -> &Self::Target {
        unsafe { &*Self::PTR }
    }
}
}

实现要点：

  • Periph 具有一个 PTR 常量，表示寄存器块的指针
  • 数值为 A，即 pub type Gpioa = crate::Periph<gpioa::RegisterBlock, 0x4800_0000>; 中的 0x4800_0000
  • core::ops::Deref 实现了 Deref trait，使得 Periph 解引用时自动转化为 RB 结构体
  • 结构体的基地址为 A

操作寄存器

寄存器读写权限

获取到寄存器对应的字段后，需要对其进行读写操作。在 C/C++ 中，这类语言无法使用静态检查避免误操作，但 Rust 语言可以。

ahbenr() 函数分析

我们深入查看 ahbenr() 函数：

#![allow(unused)]
fn main() {
// crates/stm32f303pac/src/rcc.rs
#[doc = "0x14 - AHB Peripheral Clock enable register (RCC_AHBENR)"]
#[inline(always)]
pub const fn ahbenr(&self) -> &Ahbenr {
    &self.ahbenr
}
}

说明：

  • crates/stm32f303pac/src/rcc.rs 文件包含了 RCC 外设的所有寄存器
  • Ahbenr 寄存器的地址相对于基地址偏移 0x14
  • 位于结构体 rcc::RegisterBlock 中的第五个成员

Reg 结构体

Ahbenr 寄存器类型为：pub type Ahbenr = crate::Reg<ahbenr::AhbenrSpec>;

使用 crate::Reg<> 类型：

#![allow(unused)]
fn main() {
// crates/stm32f303pac/src/generic.rs
pub struct Reg<REG: RegisterSpec> {
    register: vcell::VolatileCell<REG::Ux>,
    _marker: marker::PhantomData<REG>,
}
}

字段说明：

  • register 字段：VolatileCell 类型，用于存储寄存器的值，读写时不会被优化
  • _marker 字段：PhantomData 类型，用于编译时检查寄存器类型，不占用内存空间
  • REG::Ux 是一个关联类型，指定寄存器的数据类型

RegisterSpec trait

#![allow(unused)]
fn main() {
// crates/stm32f303pac/src/generic.rs
pub trait RegisterSpec {
    #[doc = " Raw register type (`u8`, `u16`, `u32`, ...)."]
    type Ux: RawReg;

AhbenrSpec 类型实现

查看文件 crates/stm32f303pac/src/rcc/ahbenr.rs，此文件为 AhbenrSpec 类型进行了多种包装：

#![allow(unused)]
fn main() {
// 位数为32位
impl crate::RegisterSpec for AhbenrSpec {
    type Ux = u32;
}

// 设置为可读
#[doc = "`read()` method returns [`ahbenr::R`](R) reader structure"]
impl crate::Readable for AhbenrSpec {}

// 设置为可写，但写操作是 Unsafe 的
#[doc = "`write(|w| ..)` method takes [`ahbenr::W`](W) writer structure"]
impl crate::Writable for AhbenrSpec {
    type Safety = crate::Unsafe;
}

// 设置默认值为0x14
#[doc = "`reset()` method sets AHBENR to value 0x14"]
impl crate::Resettable for AhbenrSpec {
    const RESET_VALUE: u32 = 0x14;
}
}

实现要点：

  • 完整地描述了 Ahbenr 寄存器的各个位域、初始值、读写限制等
  • 使用 R 与 W 包装 AhbenrSpec，分别设置位域的读写权限

modify 函数详解

函数签名

#![allow(unused)]
fn main() {
pub fn modify<F>(&self, f: F) -> REG::Ux
}

参数说明

  • 闭包参数 f：接受一个闭包参数用于修改寄存器
  • 返回值：寄存器的最终值，类型为 REG::Ux（通常是 u32 等整数类型）

闭包约束

#![allow(unused)]
fn main() {
for<'w> F: FnOnce(&R<REG>, &'w mut W<REG>) -> &'w mut W<REG>
}
  • 闭包接收两个参数：读取器 &R<REG> 和写入器 &'w mut W<REG>
  • 返回修改后的写入器引用

完整实现

#![allow(unused)]
fn main() {
// crates/stm32f303pac/src/generic.rs
impl<REG: Readable + Writable> Reg<REG> {
    pub fn modify<F>(&self, f: F) -> REG::Ux
    where
        for<'w> F: FnOnce(&R<REG>, &'w mut W<REG>) -> &'w mut W<REG>,
    {
        let bits = self.register.get();
        let value = f(
            &R {
                bits,
                _reg: marker::PhantomData,
            },
            &mut W {
                bits: bits & !REG::ONE_TO_MODIFY_FIELDS_BITMAP | REG::ZERO_TO_MODIFY_FIELDS_BITMAP,
                _reg: marker::PhantomData,
            },
        )
        .bits;
        self.register.set(value);
        value
    }
}
}

执行流程

  1. 读取当前值：let bits = self.register.get()
  2. 创建读写器实例：创建读取器 R 和写入器 W 实例
  3. 应用位掩码：写入器初始化时应用位掩码 
    #![allow(unused)]
fn main() {
bits & !ONE_TO_MODIFY_FIELDS_BITMAP | ZERO_TO_MODIFY_FIELDS_BITMAP
}
    这确保某些特殊位按预期处理（如清零位或置位位）
  4. 调用用户闭包：调用用户提供的闭包 f 进行修改
  5. 提取新值：从修改后的写入器中提取位值 .bits
  6. 写入寄存器：将新值写入寄存器 self.register.set(value)
  7. 返回结果：返回写入的值

安全性保证

这个方法要求 REG 实现了 Readable 和 Writable，而 AhbenrSpec 实现了这两个 trait，因此可以安全地使用 modify 方法。

闭包讲解

下面这段代码中，w为W<AhbenrSpec>,crates/stm32f303pac/src/rcc/ahbenr.rs文件为此数据类型添加了很多方法，包括iopeen(),iopeen()返回crate::BitWriter<'a, REG>类型，具有set_bit()和clear_bit()方法。

// crates/stm32f303pac/src/generic.rs
impl<'a, REG, FI> BitWriter<'a, REG, FI>
where
    REG: Writable + RegisterSpec,
    bool: From<FI>,
{
    #[doc = " Sets the field bit"]
    #[inline(always)]
    pub fn set_bit(self) -> &'a mut W<REG> {
        self.w.bits |= REG::Ux::ONE << self.o;
        self.w
    }
    #[doc = " Clears the field bit"]
    #[inline(always)]
    pub fn clear_bit(self) -> &'a mut W<REG> {
        self.w.bits &= !(REG::Ux::ONE << self.o);
        self.w
    }
}

clear_bit，set_bit需要REG实现Writable和RegisterSpectrait，而AhbenrSpec是实现了的，因此可以调用。

汇编代码逐行解释

函数入口和栈帧设置

08000468 <enable_gpioe_clock>:
 8000468: b580         	push	{r7, lr}
 800046a: 466f         	mov	r7, sp
  • 地址 0x08000468：函数 enable_gpioe_clock 的入口地址
  • push {r7, lr}：将寄存器 r7（帧指针）和 lr（链接寄存器/返回地址）压入栈中，保存调用者的上下文
  • mov r7, sp：将当前栈指针 sp 的值复制到 r7，建立新的栈帧

构造寄存器地址

 800046c: f241 0014    	movw	r0, #0x1014
 8000470: f2c4 0002    	movt	r0, #0x4002
  • movw r0, #0x1014：将立即数 0x1014 加载到寄存器 r0 的低16位
  • movt r0, #0x4002：将立即数 0x4002 加载到寄存器 r0 的高16位
  • 组合结果：r0 = 0x40021014，这是 RCC_AHBENR 寄存器的地址

读取-修改-写入操作

 8000474: 6801         	ldr	r1, [r0]
 8000476: f441 1100    	orr	r1, r1, #0x200000
 800047a: 6001         	str	r1, [r0]
  • ldr r1, [r0]：从地址 r0（0x40021014）读取32位数据到寄存器 r1
  • orr r1, r1, #0x200000：将 r1 与立即数 0x200000（bit 21）进行按位或运算
    • 0x200000 = 2^21，对应 IOPEEN 位（GPIO Port E 时钟使能位）
  • str r1, [r0]：将修改后的值 r1 写回到地址 r0

函数返回

 800047c: bd80         	pop	{r7, pc}
  • pop {r7, pc}：从栈中恢复 r7 和 pc（程序计数器）
  • 恢复调用者的栈帧，并跳转到返回地址继续执行

分析总结

芯片外设rcc的地址为0x40021000，寄存器ahbenr偏移为0x14。因此，寄存器ahbenr的地址为0x40021014。

这段汇编代码实现了一个典型的读取-修改-写入（Read-Modify-Write）操作：

  1. 目标寄存器：RCC_AHBENR (0x40021014)
  2. 操作位：bit 21 (IOPEEN)
  3. 功能：使能 GPIO Port E 的时钟

零成本抽象

零成本抽象（Zero-Cost Abstraction）是指在使用抽象接口时，编译器能够将抽象接口的调用转换为与之等价的低级代码，从而实现零成本的抽象。这种抽象方式可以提高代码的可读性和可维护性，同时保持高性能。

在如何控制寄存器中，程序使用了Rust语言的零成本抽象特性来描述芯片外设的硬件限制，例如寄存器的位宽、地址、读写限制等。通过这种方式，程序可以更清晰地表达硬件的限制，无需担心开发者错误操作硬件，从而保证了程序的正确性和可靠性。

在反汇编文件中，可以清晰地看到，Rust的各种操作限制最终会被转换为没有任何副作用的指令，汇编码中并不存在任何与操作限制相关的指令，而只是对特定寄存器直接进行读写操作。

这便是零成本抽象的体现，它使得程序在使用抽象接口时，能够享受到高度抽象、类型安全的优势，同时又能达到C/C++这类系统语言的性能。
